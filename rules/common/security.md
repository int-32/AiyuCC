# 安全规范

安全是不可妥协的底线。每次提交前必须检查以下清单。

## 提交前安全检查清单

1. **无硬编码密钥** — API keys, tokens, passwords, JWT secrets 必须使用环境变量
2. **参数化查询** — SQL 查询禁止字符串拼接，必须使用参数化绑定
3. **XSS 防护** — 用户输入展示前必须转义，使用 textContent 而非 innerHTML
4. **CSRF 防护** — 所有状态变更请求需要 CSRF token
5. **认证检查** — 每个受保护路由必须验证认证状态
6. **速率限制** — 所有公开端点必须有速率限制
7. **错误信息** — 面向用户的错误不泄露内部实现（堆栈、SQL、路径）

## 强制规则

- 发现高危安全问题时**立即停止**，不要继续其他工作
- 绝不在代码、日志、注释中出现明文密钥
- 依赖更新前检查已知漏洞
- 输出前检查是否包含敏感数据（脱敏处理）
