# 编码风格规范

## 不可变性

始终优先使用不可变数据结构，返回新副本而非就地修改。
- Python: `frozen=True` dataclass, tuple, `dict | spread` 替代直接修改
- TypeScript: `const`, `readonly`, spread 操作符

## 文件组织

- 单文件 200-400 行常规，**800 行绝对上限**
- 每个文件单一职责
- 多个小文件优于少数大文件

## 函数设计

- 函数不超过 50 行
- 嵌套不超过 4 层 — 使用 early return
- 参数不超过 5 个，超过时使用配置对象
- 命名清晰表达意图

## 错误处理

- 每层有适当的错误处理，不吞掉错误
- 使用类型化错误（自定义 Exception / Error 类）
- 面向用户的错误不泄露内部实现
- fail-fast: 函数入口验证输入

## 命名

- 变量名、函数名使用英文
- 描述性命名，避免无意义缩写
- 布尔值 `is_`/`has_`/`should_` 开头
- 常量 UPPER_SNAKE_CASE

## 注释

- 代码自解释，注释解释 "为什么" 而非 "是什么"
- 不注释掉代码 — 删除它
- 不使用 emoji
