# /update-docs - 文档更新

检查并更新与代码变更相关的文档。

## 执行步骤

1. 运行 `git diff` 查看代码变更
2. 识别受影响的文档:
   - README.md（项目说明、使用方法）
   - API 文档（端点变更）
   - 配置文档（新增配置项）
   - CHANGELOG.md（版本变更记录）
3. 更新文档使其与代码一致
4. 不创建多余的文档文件

## 用户参数

$ARGUMENTS

支持:
- 无参数: 自动检测需要更新的文档
- 文件路径: 更新指定文档
