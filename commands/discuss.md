# /discuss - 需求讨论

使用 discuss-facilitator agent 进行结构化的需求讨论，在项目启动前确保需求清晰完整。

## 执行步骤

1. **背景确认** — 引导用户描述项目背景和动机
   - 反问: 追问触发点、用户画像、现有痛点 (最多 3 题)
2. **目标澄清** — 确认核心目标、非目标、成功标准
   - 反问: 追问优先级、边界、量化指标 (最多 3 题)
3. **IO 示例** — 收集具体的输入数据和期望输出示例
   - 反问: 追问边界情况、异常处理、格式偏好 (最多 3 题)
4. **完整性检查** — 汇总模糊点，一次性请用户确认
5. **开源调研** — 搜索至少 5 个类似开源项目（按 Star 和更新时间排序）
6. **生成文档** — 将讨论内容整理保存到 `.claude/discussions/<topic>.md`

## 用户参数

$ARGUMENTS

支持:
- 项目/功能名称，如 `/discuss 用户认证系统`
- 无参数时，先询问要讨论的主题

## 调用 Agent

使用 Task 工具调用 discuss-facilitator agent:
- subagent_type: Discuss
- 提供用户描述的初始需求
- model: opus

## 反问机制

每个阶段结束后，agent 会主动追问:
- 每轮最多 **3 个问题**
- 问题具体有方向，带选项降低回答难度
- 用户说「没有补充」立即进入下一阶段
- 所有问答都记录到最终文档

## 开源项目调研

使用 WebSearch 搜索类似项目:
- 从项目背景和目标中提取搜索关键词
- 最少 5 个项目
- 收集: 项目名、URL、Star 数、最近更新时间、简述
- 分析与当前需求的相关度和可借鉴点
- 按 Star 数降序排序

## 输出

生成讨论文档到 `.claude/discussions/<topic>.md`，包含:
- 项目背景（关键信息提炼）
- 目标清单（按优先级排序）
- IO 示例表（含边界和异常情况）
- 开源项目参考表
- 完整讨论记录（Q&A 格式）
- 待确认事项和下一步建议

## 后续步骤

文档生成后，建议用户:
1. 审阅讨论文档，补充遗漏
2. 运行 `/plan` 基于讨论文档创建实施计划
