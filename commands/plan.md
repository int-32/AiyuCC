# /plan - 实施规划

使用 planner agent 创建详细的实施计划，然后拆分为可执行的任务清单。

## 执行步骤

1. **重述需求** — 用自己的话重述用户需求，确认理解正确
2. **分析代码库** — 使用 Glob/Grep/Read 了解相关代码结构
3. **模块拆分** — 将需求拆解为独立模块/组件，明确边界和接口
4. **创建计划** — 分阶段、有依赖关系、有风险评估的实施计划
5. **注册任务** — 用 TodoWrite 将计划中的步骤注册为可追踪的任务清单
6. **等待确认** — 不要在用户确认前写任何代码

## 用户参数

$ARGUMENTS

如果用户未提供参数，询问要规划的功能。

## 调用 Agent

使用 Task 工具调用 planner agent:
- subagent_type: Plan
- 提供完整的上下文和需求描述
- model: opus

## 后续步骤

planner agent 返回计划后:

1. **保存计划** — 将完整计划写入 `.claude/plans/<plan-name>.md`（持久化，跨会话可用）
2. **提取任务清单** — 从计划中提取所有具体步骤，注意 Wave 和并行组标记
3. **注册 TodoWrite** — 将步骤按 Wave 顺序注册为 todo 项，格式:
   - content: "Wave2: 步骤描述 (文件路径) [⚡并行-A]"
   - activeForm: "正在执行: 步骤描述"
   - status: pending
4. **展示给用户** — 显示完整计划（含并行依赖图和 Agent Team 调度表），等待确认
5. **按 Wave 执行** — 确认后按 Wave 顺序执行:
   - **串行 Wave**: 按顺序逐步执行
   - **并行 Wave**: 通过 Task 工具同时启动多个 Agent 并行执行同一 Wave 内的所有任务
   - 每个 Wave 全部完成后，运行受影响的测试做集成验证
   - 验证通过后再进入下一个 Wave
   - 每完成一步，标记 TodoWrite 为 completed，同步更新计划文件 `- [ ]` → `- [x]`

## Agent Team 并行执行说明

当 Wave 标记为"⚡并行执行"时:
- 使用 Task 工具为该 Wave 中的每个任务创建独立的子任务
- 每个子任务分配给 Agent Team 调度表中指定的 Agent
- 各 Agent 在独立上下文中并行工作，互不干扰
- 等待所有并行任务完成后，统一验证再进入下一 Wave

## 持久化说明

- 计划文件保存在项目的 `.claude/plans/` 目录下
- 文件名基于功能名称，如 `.claude/plans/user-auth.md`
- 每完成一个任务，用 Edit 工具将计划文件中对应的 `- [ ]` 改为 `- [x]`
- 新会话读取计划文件时，通过 checkbox 状态即可知道哪些已完成、哪些待继续
